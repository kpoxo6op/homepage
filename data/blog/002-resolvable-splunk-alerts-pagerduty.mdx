I was impressed by this [post]() on Splunk support forum explaining
how to automatically resolve PagerDuty alerts received from Splunk.

It took me some time to understand the post, so I decided to expand
it to an easy-to-follow guide to help others.

It's remarkable that it's been 4 years since the original post but
there is still no support for autoresolving Splunk alerts as per
PagerDuty [docs](https://www.pagerduty.com/docs/guides/splunk-integration-guide/)

This guide explains how to build the fully working demo.
Feel free to follow along step by step or just read the contents.

## Table of Contents

## Before you begin

Sign up for [Splunk](https://www.splunk.com/en_us/sign-up.html) and [PagerDuty](https://www.pagerduty.com/sign-up/) accounts.

Clone GitHub [repository]() with the demo code.

Install terraform, docker and docker compose.

## `.env`

Create `.env` file.

```sh
cp .env.sample .env
```

## PagerDuty Token

Create [API Token](https://support.pagerduty.com/docs/api-access-keys#generate-a-user-token-rest-api-key).

Set `PAGERDUTY_TOKEN` variable in `.env` file with the token.

## Start Local Splunk

Start local docker compose stack, wait for healthy Splunk status.

```sh
docker compose up -d
watch docker compose ps
```

## Run terraform

Run terraform to create required resources in PagerDuty and Splunk.

_NOTE_: Splunk will require some manual actions. See the next step.

## Splunk Manual Steps

Login to local Splunk as `admin/abcd1234`.

Install PagerDuty [App](http://localhost:8000/en-US/manager/search/appsremote?query=pagerduty).

Open the [search](http://localhost:8000/en-US/app/search/search?s=%2FservicesNS%2Fnobody%2Fsearch%2Fsaved%2Fsearches%2FBoris%2520As%2520Code%2520Errors) created by terraform, save it as Alert:
